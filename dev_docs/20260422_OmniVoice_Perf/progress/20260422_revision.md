## 추가 보강 — 영구 점유 (Persistent Residency) 철학 도입

사용자 추가 피드백:
> "vllm 처럼 GPU의 사용량 및 VRAM 일부를 서비스 실행과 함께 영구적으로 점유하고 있어도 괜찮으니 (서비스 꺼지기 전까지) 제대로 만들어야만해. 이런 철학까지도 반영해서 완벽하게 개선해보자."

→ **하드 제약 #4** 로 격상하여 전 문서에 반영.

### 도입 배경

omnivoice 컨테이너는 *유일한 GPU 텐션트* (단일 모델, 단일 서비스). VRAM 을 다른 프로세스와 공유할 동기 없음. → vLLM 이 KV-cache pool 을 시작 시 일괄 점유하듯, omnivoice 도 모든 동적 자원을 lifespan 에서 사전 할당하고 *해제하지 않는다*. 런타임 중 새로운 큰 할당이 0 이 되도록 만든다.

### 신설 최적화 항목 (모두 Tier-A, 출력 동치)

| ID | 내용 | 기여 |
|----|------|------|
| **O14** | `GenerationWorkspace` — `_generate_iterative` 의 `batch_input_ids` / `attention_mask` / `logits_buf` / `score_buf` / `sample_buf` 를 *최대 T_total* 기준 1회 할당, 매 호출 슬라이스 뷰로 in-place 재사용 | per-call allocator 비용 5~30ms 제거 + jitter 감소 |
| **O15** | `PinnedPCMPool` — 출력 PCM int16 버퍼를 pinned host memory 로 사전 할당, `to('cpu', non_blocking=True)` + 전용 D2H stream | per-call 1~5ms + 다음 컴퓨트와 오버랩 가능 |
| **O16** | Multi-shape warmup — short/medium/long 3 bucket × cond+uncond 1회씩 lifespan 합성 → cuDNN/cuBLAS 알고리즘 캐시 + autotune workspace 확정 | bucket-별 cold gap 0 |
| **O17** | Allocator/스트림 정책 고정 — `PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True,max_split_size_mb=128`, `set_per_process_memory_fraction(0.9)`, 보조 H2D/D2H stream lifespan 생성 | fragmentation 회피 + jitter 감소 |

### 신설 게이트 — `/diag/memory` + `check_memory_residency.py`

영구 점유 정책의 *결정적 검증*. compare_audio 가 출력 동치를 보장한다면 이것은 *메모리 동맥* 을 보장.

| 지표 | 허용 |
|------|------|
| `allocated_bytes.all.current` 증가 | 0 (±0.5%) |
| `reserved_bytes.all.current` 증가 | 0 |
| `num_alloc_retries` 증가 | 0 |
| `num_ooms` | 0 |
| fragmentation `(reserved-allocated)/reserved` | ≤ 5% |

하나라도 깨지면 PR 머지 거부. *동적 할당이 남은 코드 경로* 가 있다는 결정적 증거.

### Phase 1d 신설 (5 sub-step)

- 1d-1: Allocator/스트림 정책 고정 (가장 먼저)
- 1d-2: `GenerationWorkspace` 사전 할당
- 1d-3: `PinnedPCMPool` 도입
- 1d-4: 3-bucket warmup (1a 의 강화판)
- 1d-5: `/diag/memory` 진단 + 회귀 게이트

### VRAM budget 명시

GTX 1070 8GB 분배:
- 모델 가중치 fp16: ~3.0 GB
- `GenerationWorkspace`: ~1.0~1.5 GB (logits_buf 가 최대)
- ref-cache 8 voice: ~0.2 GB
- cuDNN/cuBLAS workspace: ~0.5 GB
- 시스템/디스플레이 점유: ~0.5 GB
- **합계 목표 ~5.5 GB** (`OMNIVOICE_GPU_MEM_FRACTION=0.7` 안전선, 디스플레이 미사용 호스트는 0.9 까지)

### KPI 변경

- ❌ `GPU peak memory` baseline +10% 이내
- ✅ `GPU steady-state VRAM` *시작 직후 ≈ 합성 후 ≈ 종료 직전 ±5%*
- ✅ `(reserved - allocated) / reserved ≤ 5%` (단편화 0)
- ✅ p95-p50 RTF gap 감소 (jitter 안정화)
